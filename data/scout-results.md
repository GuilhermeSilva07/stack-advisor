# Resultados do Scout

## Comparações de Frontend
1. **Next.js** – Pro: SEO-friendly, server‑side rendering, file‑system routing; Contra: maior complexidade de build, tempo de compilação; Casos reais: Vercel (plataforma própria), TikTok Web, Hulu.
2. **React** – Pro: enorme comunidade, flexibilidade; Contra: requer configuração de SSR separada; Casos reais: Facebook, Instagram, Airbnb.

## Comparações de Backend
1. **Node.js + Express** – Pro: JavaScript full‑stack, fácil de iniciar, grande ecossistema; Contra: single‑threaded, performance limitada em CPU‑intenso; Casos reais: LinkedIn Mobile API, PayPal (micro‑services).
2. **NestJS** – Pro: arquitetura modular, TypeScript nativo, suporte a micro‑services; Contra: curva de aprendizado maior; Casos reais: Adidas e-commerce, Rocket.Chat.

## Comparações de Banco de Dados
1. **PostgreSQL (Supabase)** – Pro: SQL avançado, suporte a JSON, escalável, gratuito no tier; Contra: requer gerenciamento de migrações; Casos reais: GitLab, Reddit (parte da stack).
2. **MySQL** – Pro: amplamente usado, fácil de hospedar; Contra: menos recursos avançados que PostgreSQL; Casos reais: WordPress, Uber (legacy).

## Comparações de Deploy
1. **Vercel** – Pro: integração zero‑config com Next.js, preview builds automáticas; Contra: limites de uso na camada gratuita; Casos reais: Next.js docs, Hashnode.
2. **Railway** – Pro: deploy rápido de full‑stack, variáveis de ambiente simples; Contra: menos controle de infra‑estrutura avançada; Casos reais: startups early‑stage, projetos open‑source.
